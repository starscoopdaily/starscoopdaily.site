## 4. Article Writing Style & Tone

### Voice
- **Conversational but credible** — not tabloid gossip, not dry Wikipedia
- Write like you're telling your most interested friend who follows Hollywood obsessively
- Always from a fan perspective — excited, curious, not cynical or snarky
- British Royals and Fashion: slightly more polished, less casual
- Ending Explained and Where to Watch: more structured and informative

### Paragraph Rules
- **Max 2–3 sentences per paragraph** — white space = mobile readability
- Never write walls of text — break every thought into its own paragraph
- Use **bold** for names, numbers, key facts on first mention in a paragraph
- No passive voice — say "Kate wore the dress" not "the dress was worn by Kate"

### Headline Formula
Use one of these proven headline structures:

```
[Celebrity] Just [Did Something] — Here's What We Know
Is [Celebrity] Really [Claim]? The Truth Revealed
[Movie/Show] Ending Explained — What Really Happened
Everything We Know About [Title] Before [Date]
[Celebrity]'s [Event] — Full Breakdown
Why [Celebrity] [Did X] — The Real Reason
[Number] Things [Movie] Got Wrong About [Topic]
[Movie] Review — Worth Watching? (Spoiler-Free)
```

### Article Structure (every article)
```
[Intro — 2–3 sentences, hook + main claim]

[H2: First main point]
[2–3 paragraphs]

[H2: Second main point]
[2–3 paragraphs]
[Inline image after 2nd H2]

[H2: Third main point]
[2–3 paragraphs]

[H2: Fourth main point]
[2–3 paragraphs]
[Inline image after 4th H2]

[H2: What Does This Mean / Conclusion]
[2–3 paragraphs, close with what to watch/follow next]
```

### Word Count
- Breaking news / reaction pieces: 400–600 words
- Relationship / drama pieces: 600–900 words
- Ending Explained: 800–1200 words
- Where to Watch / evergreen guides: 700–1000 words
- Reviews: 600–900 words

### What to Avoid
- Saying "In conclusion" or "To summarize" — just end naturally
- Clickbait without payoff — headline must match content
- Unverified rumors stated as facts — say "reportedly" or "sources say"
- Copying from other outlets — every sentence must be original
- Filler phrases: "It goes without saying", "Needless to say", "At the end of the day"

## 5. Image Placement Rules

### Hero Image
| Content Type | Image to Use | Display Style |
|---|---|---|
| Movie | TMDB backdrop (landscape, 1280px) | Full-bleed cover |
| TV Show | TMDB backdrop | Full-bleed cover |
| Celebrity / Person | TMDB profile photo | Portrait + blur backdrop |
| British Royals | TMDB profile photo or manual URL | Portrait + blur backdrop |
| No TMDB match | Pexels keyword search | Cover or contain |

**Hero never:**
- Base64 images (huge file size)
- Wikipedia URLs (breaks next/image)
- Blurry or low-resolution images

### Inline Body Images (injected after 2nd and 4th H2)
- **Landscape images:** full column width (100%), no centering
- **Portrait images:** 85% width, centered, max-width 560px
- Use `makeFigure` in admin to inject — never manually paste base64

### Image Sources (priority order)
1. TMDB API (movie backdrops, posters, person profile photos) — always first
2. OMDb poster (fallback for movies)
3. Manual URL from verified source (official press kits, studio sites)
4. Pexels keyword search (generic/mood images only)

## 6. SEO Strategy

### Keyword Research — Do This Before Every Article

**Step 1: Check Google Trends**
- Go to trends.google.com
- Search the celebrity/movie/show name
- Check "Interest over time" — is it rising or falling?
- Look at "Related queries" for article angle ideas

**Step 2: Check Google News**
- Go to news.google.com
- Search the topic
- See what's being covered — find an angle nobody has written yet
- Check the timestamps — if breaking news is under 2 hours old, publish fast

**Step 3: Check "People Also Ask" on Google**
- Search your topic on Google
- Screenshot all "People Also Ask" questions
- Answer at least 2–3 of them inside your article as H2 headings
- This is how you get featured snippets

**Step 4: Long-tail keyword targeting**
- Never target just "[Celebrity Name]" — too competitive
- Always target: "[Celebrity Name] + [specific thing]"
  - Examples: "kate middleton cancer update 2026", "sabrina carpenter boyfriend 2026", "obsession movie ending explained"
- Put the main keyword in: Title, First H2, Meta description, Image alt text, First paragraph

**Step 5: Answer The Public (free)**
- Go to answerthepublic.com
- Enter the topic
- Find "how", "why", "what", "when" questions
- Use these as H2 subheadings in your article

### On-Page SEO Checklist (every article)
- [ ] Keyword in title (within first 60 characters)
- [ ] Keyword in meta description (under 155 characters)
- [ ] Keyword in first paragraph
- [ ] At least 3 H2 subheadings with related keywords
- [ ] All images have descriptive alt text (not "image1.jpg")
- [ ] Internal links to 2–3 related articles on the site
- [ ] Article length appropriate for the topic (see Section 4)
- [ ] Date is accurate and will auto-update for freshness

### Topic Freshness Rules
- **Evergreen articles** (Where to Watch, character explainers): update them every 2–3 months with fresh info
- **Breaking news**: publish within 2 hours or don't bother — Google rewards freshness
- **Ending Explained**: publish on release day or day-after — the search spike is 72 hours long

## 9. Content Quality Rules (Non-Negotiable)

1. **Never publish unverified gossip as fact** — use "reportedly", "sources say", "according to X"
2. **Always attribute** — if info comes from an interview, name the outlet
3. **No AI-obvious phrases** — remove: "In this article", "Let's dive in", "It's worth noting", "Delve into"
4. **Personalize before publishing** — add one original opinion or observation per article
5. **Proofread the headline twice** — that's the only thing most readers see
6. **Check that all images load** before saving the article
7. **Verify TMDB data** — double check cast, release dates, ratings in the article match TMDB
8. **Don't publish stubs** — minimum 400 words, all H2s must have content

## 12. API Calls — What Gets Called & When

Every article generation triggers these APIs in order:

| Step | API | What It Does | When |
|---|---|---|---|
| 1 | **Claude AI** (admin → `/api/generate`) | Generates full article HTML from topic + category | On "Generate Article" click |
| 2 | **TMDB Search** (`/api/tmdb?type=movie/tv/person&query=`) | Searches for matching movie, TV show, or person | Auto after article generates |
| 3 | **TMDB Details** (`/api/tmdb?type=movie/tv/person&id=`) | Fetches full details — backdrop, poster, cast, ratings | Auto after top search result found |
| 4 | **OMDb API** (inside `/api/tmdb` server route) | Fetches IMDB rating, Rotten Tomatoes %, Metacritic score | Inside TMDB movie details call |
| 5 | **Wikipedia REST API** (inside `/api/tmdb` server route) | Fetches person bio extract + thumbnail photo | Inside TMDB person details call |
| 6 | **Pexels API** (`/api/pexels?query=`) | Fetches stock images if no TMDB image available | Manual — image picker fallback |
| 7 | **GitHub API** (admin → `saveConfig` / publish) | Saves article JSON to repo → triggers Vercel rebuild | On "Publish" click |

**Environment variables required:**
- `TMDB_API_KEY` — Vercel env var (never in code)
- `OMDB_API_KEY` — Vercel env var
- `PEXELS_API_KEY` — Vercel env var
- GitHub token — stored in admin localStorage (Site Controls tab)
